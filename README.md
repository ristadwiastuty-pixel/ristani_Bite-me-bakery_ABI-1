## Model Bisnis & Aliran Pendapatan
 
**Model Bisnis:** Direct-to-Consumer (D2C) melalui website sendiri, dikombinasikan dengan penjualan offline di toko fisik.
 
**Aliran Pendapatan (Revenue Streams):**
1. **Penjualan produk reguler** — roti, kue, dan snack harian.
2. **Pesanan custom/pre-order** — kue ulang tahun, hampers acara khusus dengan DP (down payment).
3. **Paket bundling** — kombinasi produk dengan harga lebih menarik dibanding beli satuan.
4. **Layanan katering ringan** — snack box untuk kebutuhan korporat/acara.
5. **Biaya pengiriman (delivery fee)** — untuk pesanan yang dikirim di luar radius gratis ongkir.

## Dokumentasi Teknis

### Ringkasan Teknologi

Website ini dibangun sebagai **single-page application (SPA)** statis menggunakan:

- **HTML5** — struktur halaman
- **CSS3** (murni, tanpa framework) — styling, layout responsif, animasi transisi
- **JavaScript (Vanilla JS)** — logika interaktif (keranjang, slider, pencarian, modal, dsb.)
- **Google Fonts**: `Baloo 2` (judul, playful) dan `Quicksand` (teks isi)
- Tidak menggunakan backend/server maupun database — seluruh data (keranjang, form) berjalan di sisi client (browser) selama sesi berlangsung
- Gambar produk & QR code pembayaran ditanam langsung ke dalam kode dalam format **base64**, sehingga file tetap satu kesatuan tanpa folder aset terpisah

### Fitur Website

- ✅ Katalog produk berbentuk bulat dengan efek "bite mark" khas branding
- ✅ Rating bintang & ulasan pelanggan di setiap produk
- ✅ Badge **Best Seller** dan deskripsi singkat untuk produk unggulan
- ✅ Hero banner (slider) untuk produk rekomendasi, bisa digeser & diklik langsung ke katalog
- ✅ Search bar dengan auto-scroll & highlight ke produk yang dicari
- ✅ Keranjang belanja (tambah, ubah jumlah, hapus, reset)
- ✅ Form pemesanan (nama, alamat, username Instagram, no HP)
- ✅ 4 metode pembayaran: Cash, QRIS (QR code asli toko), Transfer BNI, E-Wallet DANA
- ✅ Popup rincian pembayaran otomatis (nomor rekening/QR + total tagihan)
- ✅ Upload bukti pembayaran + kirim konfirmasi otomatis ke WhatsApp toko
- ✅ Navigasi kategori di sidebar kiri
- ✅ Bagian kontak (WhatsApp, Email, Instagram, Alamat) yang bisa langsung diklik
- ✅ Tampilan responsif penuh untuk smartphone, tablet, dan desktop

## Proses Checkout & Simulasi Payment Gateway
 
**Alur Checkout:**
1. Pelanggan memilih produk dari katalog → masuk ke keranjang (cart).
2. Pelanggan mengisi data pengiriman (nama, alamat, nomor kontak) dan memilih metode pengambilan (pickup/delivery).
3. Sistem menampilkan ringkasan pesanan (subtotal, ongkos kirim, total pembayaran).
4. Pelanggan memilih metode pembayaran dan melanjutkan ke halaman pembayaran.
5. Setelah pembayaran berhasil, pelanggan menerima konfirmasi pesanan (nomor order & estimasi waktu selesai).
**Simulasi Payment Gateway — Dummy Midtrans:**
Untuk kebutuhan demo/prototipe, proses pembayaran disimulasikan menggunakan pendekatan seperti **Midtrans Snap** (dummy, tanpa transaksi nyata):
 
```javascript
// Contoh simulasi dummy Midtrans di JavaScript.JS
function simulateMidtransPayment(order) {
  console.log("Memulai simulasi pembayaran Midtrans...");
  console.log("Detail pesanan:", order);
 
  // Simulasi status pembayaran (dummy, tidak terhubung ke API asli)
  const paymentStatus = Math.random() > 0.1 ? "success" : "failed";
 
  setTimeout(() => {
    if (paymentStatus === "success") {
      alert(`Pembayaran berhasil! Order ID: ${order.id}`);
    } else {
      alert("Pembayaran gagal, silakan coba lagi.");
    }
  }, 1500);
}
```

## Rencana SEO, Keamanan, dan Pemeliharaan
 
**Rencana SEO:**
- Optimasi meta title & meta description pada `index.html` dengan kata kunci relevan (misal: "toko roti online", "kue custom terdekat").
- Struktur heading (H1–H3) yang rapi dan deskriptif pada tiap kategori produk.
- Gambar produk diberi *alt text* deskriptif untuk mendukung pencarian gambar.
- Kecepatan loading halaman dioptimalkan (kompresi gambar, minifikasi CSS/JS).
- Pembuatan konten blog seputar resep/tips bakery untuk menarik trafik organik.
**Rencana Keamanan:**
- Menggunakan HTTPS untuk seluruh trafik website.
- Validasi input pada form checkout untuk mencegah *injection* atau data tidak valid.
- Tidak menyimpan data kartu pembayaran di sisi front-end — seluruh proses sensitif diarahkan ke payment gateway resmi (Midtrans).
- Backup data pesanan secara berkala.
**Rencana Pemeliharaan (Maintenance):**
- Update katalog produk & harga secara rutin.
- Pengecekan tautan, gambar, dan fungsi checkout secara berkala agar tidak ada *broken link/feature*.
- Monitoring performa website (uptime, kecepatan loading).
- Iterasi desain UI/UX berdasarkan masukan pelanggan.

## Rencana Penggunaan Data Analitik untuk Pengambilan Keputusan
 
Data yang dikumpulkan dari interaksi pelanggan di website akan digunakan untuk mendukung keputusan bisnis, antara lain:
 
- **Produk terlaris & kurang laku** → menyesuaikan produksi harian agar mengurangi *food waste* dan memastikan stok produk favorit selalu tersedia.
- **Waktu puncak transaksi (peak hours)** → menentukan jadwal produksi dan jam operasional/delivery yang optimal.
- **Asal & perilaku pengunjung website** (halaman yang sering dikunjungi, tingkat *bounce rate*) → menyempurnakan tampilan katalog dan alur checkout.
- **Tingkat konversi (visitor → checkout)** → mengevaluasi efektivitas promosi dan desain halaman produk.
- **Riwayat pembelian pelanggan** → dasar untuk program loyalitas dan rekomendasi produk personal (*personalized recommendation*).
- **Feedback & rating pelanggan** → dasar evaluasi kualitas produk dan layanan secara berkelanjutan.
Data-data ini dapat dikumpulkan melalui alat seperti Google Analytics (trafik website) dan pencatatan transaksi internal, lalu dievaluasi secara berkala (mingguan/bulanan) untuk mendukung keputusan strategis bisnis Bite Me Bakery.
